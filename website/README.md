# claude-desktop-site

Static landing page for the community Claude Desktop build for Linux.
Next.js 14 (App Router) + Tailwind, exported as plain HTML/CSS via
`output: 'export'`.

## Run locally

```bash
cd website
npm install
npm run dev        # → http://localhost:4000
```

## Build static site

```bash
npm run build
# → out/  (drop onto GitHub Pages, Netlify, Cloudflare Pages, S3, …)
```

The build is fully static. `out/index.html` is the landing page; every
font and style file is baked into `out/_next/`. `out/sitemap.xml` and
`out/robots.txt` are generated by the file-based conventions below.

## SEO

Everything SEO-related lives in `lib/seo.js`. Change one constant,
everything downstream updates: metadata tags, canonical, OG, Twitter
card, JSON-LD, sitemap, robots.

- **`lib/seo.js`** — central config:
  - `SITE_URL` · `SITE_NAME` · `SITE_TITLE` · `SITE_DESCRIPTION`
  - `SITE_KEYWORDS` — keyword list (Claude Desktop Linux, Ubuntu,
    Fedora, Arch, Debian, Gentoo, Cowork, bubblewrap, etc.).
  - `softwareApplicationJsonLd()` — schema.org `SoftwareApplication`
    blob with download URL, version, feature list, platform.
  - `faqJsonLd(faqs)` — schema.org `FAQPage` blob built from the FAQ
    array in `page.jsx`.
- **`app/layout.jsx`** — Next 14 metadata API: title template,
  description, keywords, canonical, robots directives, Open Graph,
  Twitter card, icons, theme colour. Injects the
  `SoftwareApplication` JSON-LD into `<head>`.
- **`app/page.jsx`** — visible H1/H2s carrying the target keywords
  (“Claude Desktop for Linux”, “sandboxed by default via bubblewrap”,
  distro names). Built-in FAQ block, mirrored into `FAQPage` JSON-LD so
  Google can render it as a rich-result.
- **`app/robots.js`** — generates `/robots.txt` (allow all, points to
  sitemap).
- **`app/sitemap.js`** — generates `/sitemap.xml`.

> **Change the deployed URL** in one place: `SITE_URL` in
> `lib/seo.js`. Default is `https://janithcooray.github.io/claude-desktop`.

## Theme

- **Colours** — orange accent (`ember.*`, primary `#ff6a1a`) over
  near-black carbon greys. No purple.
- **Fonts** — [Chakra Petch](https://fonts.google.com/specimen/Chakra+Petch)
  for display (angular, slightly cyber-techno),
  [JetBrains Mono](https://fonts.google.com/specimen/JetBrains+Mono)
  for chrome / code.
- **Motion** — animated grid background, pulsing CTA glow, blinking
  alpha indicator. Pure CSS; no JS runtime animation.

## What to tweak

- `app/page.jsx` — all visible content, plus the `FAQS` array at the
  top (edit once → both visible FAQ and `FAQPage` JSON-LD update).
- `lib/seo.js` — all metadata, links, keywords, JSON-LD.
- `tailwind.config.js` — palette + font tokens.
- `app/globals.css` — animations, scanlines, grid backdrop.
